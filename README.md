# chiptune--table-timer2.0
## 2026-07 安全保存改造
- 新增 `js/safe-state.js`。
- 桌位保存会在 Firestore 事务中，仅合并相对上次服务器快照发生变化的桌位/字段，避免旧 iPad 覆盖其他桌位。
- 续时及撤回使用 Firestore 事务，以服务器最新值为基础增减 1 小时。
- 页面右下角显示“已同步 / 正在保存 / 离线 / 保存失败”。离线时禁止续时及撤回。
- 尝试写入 `operationLogs` 集合作为操作审计；若现有 Firestore 规则不允许，不会影响主要保存。

## 2.2 本地优先同步说明

- 主计时页面的桌位状态会先写入 iPad 的 IndexedDB，本机保存成功后立即生效。
- 断网时仍可开始、暂停、继续、续时、撤回和结账；右下角会显示待上传数量。
- 网络恢复后，待同步操作会自动按顺序上传 Firestore。
- Firestore 同时启用了持久化缓存，作为第二层离线保护。
- 计时页面始终优先保留本机桌位状态；预约、老板设置等云端修改在联网后合并。
- 请不要清除 Safari 网站数据，否则会删除尚未上传的本机数据。
